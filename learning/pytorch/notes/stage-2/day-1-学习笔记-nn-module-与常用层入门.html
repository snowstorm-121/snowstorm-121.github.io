Day 1 学习笔记：nn.Module 与常用层入门

学习目标：AI 方向求职

本阶段重点：从第 3 周的 纯 Tensor 手写线性回归 与 手写 Softmax 分类，过渡到 PyTorch 的 nn.Module、常用层、官方 loss 和 optimizer。今天不是为了“只会调 API”，而是理解：手写的 w / b / forward / loss / backward / SGD / zero_grad，在 nn 模块体系里分别被封装成了什么。

今天主线：nn.Module + 常用层。核心心智模型是：nn.Module 不是替你跳过底层，而是把你已经手写过的参数、前向计算、梯度计算和参数更新组织成工程上更可维护的结构。

〇、贯穿全天的五条心智主线

1. nn.Module 解决的是模型组织问题

第 3 周 手写线性回归时，模型大概是：

w = torch.normal(0, 0.01, size=(num_inputs, 1), requires_grad=True)
b = torch.zeros(1, requires_grad=True)

def net(X):
    return X @ w + b

这里有三个核心部分：

参数：w, b
前向计算：X @ w + b
训练时要更新的对象：[w, b]

模型小时，这样写非常清楚。模型一复杂，比如变成：

X -> Linear1 -> ReLU -> Linear2 -> output

如果继续手写，就要自己维护：

W1, b1, W2, b2
params = [W1, b1, W2, b2]

参数多了以后容易漏，结构也分散。

nn.Module 的作用就是把这些东西组织起来：

nn.Module = 参数容器 + 前向计算规则 + 子层管理系统

它不改变训练本质，只是把训练流程工程化。

2. __init__ 定义“有什么层”，forward 定义“数据怎么走”

最小模型类：

import torch
from torch import nn


class LinearRegression(nn.Module):
    def __init__(self):
        super().__init__()
        self.linear = nn.Linear(2, 1)

    def forward(self, X):
        return self.linear(X)

这里的分工是：

__init__：创建层、参数、模型组件
forward：描述输入数据如何经过这些组件

这句话非常重要：

可学习层应该在 __init__ 里创建一次
forward 里只使用这些层做计算

不要在 forward 里新建 nn.Linear，否则每次前向传播都会重新随机初始化一组参数，训练就失去意义。

3. nn.Linear 封装的是线性变换

nn.Linear(in_features, out_features) 本质上做的是：

Y = X @ weight.T + bias

如果：

linear = nn.Linear(2, 1)

那么：

linear.weight.shape = (1, 2)
linear.bias.shape   = (1,)

注意：nn.Linear 的 weight 形状是：

(out_features, in_features)

而第 3 周手写线性回归时，常写成：

w.shape = (in_features, out_features)

所以官方层内部使用的是：

X @ linear.weight.T + linear.bias

这不是新数学，只是参数保存的方向不同。

4. model.parameters() 自动收集可学习参数

手写版本：

params = [w, b]

nn.Module 版本：

model.parameters()

原因是：

self.linear = nn.Linear(2, 1)

把 linear 挂到了 self 上。PyTorch 会自动把它登记为模型的子模块，并递归找到里面的：

linear.weight
linear.bias

学习时建议多用：

for name, param in model.named_parameters():
    print(name, param.shape, param.requires_grad)

它可以检查：

哪些参数被注册了
参数 shape 是否正确
requires_grad 是否为 True
有没有层没有被 optimizer 管到

5. 官方训练循环仍然是原来的训练本质

手写训练循环的核心是：

前向计算
算 loss
backward 得到 grad
用 no_grad 更新参数
清空 grad

官方写法变成：

optimizer.zero_grad()
loss.backward()
optimizer.step()

三句话的含义：

optimizer.zero_grad()  清空旧梯度
loss.backward()        计算当前 batch 的梯度
optimizer.step()       根据 param.grad 更新参数

optimizer 不负责算梯度，梯度仍然来自 Autograd。

一、为什么需要 nn.Module

1. 手写版本的问题

手写线性回归：

w = torch.normal(0, 0.01, size=(2, 1), requires_grad=True)
b = torch.zeros(1, requires_grad=True)

def linreg(X):
    return X @ w + b

这个版本适合学习底层，但参数和计算逻辑是分散的：

w / b 在函数外面
forward 逻辑在函数里面
训练循环还要手动知道 params = [w, b]

如果模型变成多层网络，手动维护会越来越容易出错。

2. nn.Module 的心智模型

nn.Module 可以理解为：

一个会管理参数和子层的 Python 对象

它让模型拥有统一入口：

model(X)              前向计算
model.parameters()    获取参数
model.train()         切换训练模式
model.eval()          切换评估模式
model.state_dict()    保存参数状态

今天重点先掌握：

model(X)
model.parameters()

二、nn.Linear 和手写 X @ w + b

1. nn.Linear 的输入输出

linear = nn.Linear(in_features=2, out_features=1)

X = torch.tensor([
    [1.0, 2.0],
    [3.0, 4.0],
])

y_hat = linear(X)

shape 对应：

X.shape      = (2, 2)
y_hat.shape  = (2, 1)

其中：

第一个维度 2：batch_size
第二个维度 1：每个样本输出 1 个值

2. 参数 shape

print(linear.weight.shape)
print(linear.bias.shape)

结果：

torch.Size([1, 2])
torch.Size([1])

nn.Linear(784, 10) 时：

weight.shape = (10, 784)
bias.shape   = (10,)

可以理解成：

10 个输出神经元
每个输出神经元都有 784 个输入权重

3. 手写验证

linear = nn.Linear(2, 1)

with torch.no_grad():
    linear.weight[:] = torch.tensor([[2.0, 3.0]])
    linear.bias[:] = torch.tensor([1.0])

X = torch.tensor([
    [10.0, 20.0],
    [30.0, 40.0],
])

y1 = linear(X)
y2 = X @ linear.weight.T + linear.bias

y1 和 y2 一样。

第一行手算：

10 * 2 + 20 * 3 + 1 = 81

说明：

nn.Linear 只是帮我们保存 weight / bias，并在 forward 时做线性变换

三、最小 nn.Module 写法

1. 模型类模板

class MyModel(nn.Module):
    def __init__(self):
        super().__init__()
        self.layer = ...

    def forward(self, X):
        return self.layer(X)

固定理解：

__init__ 里定义模型组件
forward 里定义计算流程

2. super().__init__() 的作用

super().__init__()

表示先初始化 nn.Module 的基础能力。

如果漏写，可能导致：

子层无法正常注册
model.parameters() 拿不到参数
模型状态管理出问题

所以写自定义模型时，先形成习惯：

class Net(nn.Module):
    def __init__(self):
        super().__init__()

3. 为什么写 model(X)，不是 model.forward(X)

通常写：

y_hat = model(X)

而不是：

y_hat = model.forward(X)

因为 model(X) 会走 nn.Module 的调用流程，再进入 forward。这中间 PyTorch 可以处理 hook、训练/推理相关机制等框架能力。

日常训练中遵守：

调用模型：model(X)
定义逻辑：forward(self, X)

四、model.parameters() 与 optimizer

1. 查看模型参数

model = LinearRegression()

for name, param in model.named_parameters():
    print(name, param.shape, param.requires_grad)

可能输出：

linear.weight torch.Size([1, 2]) True
linear.bias   torch.Size([1])    True

含义：

linear.weight / linear.bias 是模型参数
requires_grad=True，说明会被 Autograd 计算梯度

2. optimizer 的角色

optimizer = torch.optim.SGD(model.parameters(), lr=0.03)

optimizer 保存的是参数引用。

训练时：

loss.backward()
optimizer.step()

loss.backward() 会把梯度写到：

model.linear.weight.grad
model.linear.bias.grad

optimizer.step() 使用这些 .grad 更新参数。

3. 和手写 SGD 的对应关系

手写版本：

with torch.no_grad():
    for param in params:
        param -= lr * param.grad
        param.grad.zero_()

官方版本：

optimizer.zero_grad()
loss.backward()
optimizer.step()

对应关系：

params = [w, b]       -> model.parameters()
param.grad.zero_()    -> optimizer.zero_grad()
param -= lr * grad    -> optimizer.step()

五、用官方封装跑线性回归闭环

1. 数据与模型

from torch.utils.data import DataLoader, TensorDataset


def synthetic_data(w, b, num_examples):
    X = torch.normal(0, 1, size=(num_examples, len(w)))
    y = X @ w + b
    y += torch.normal(0, 0.01, y.shape)
    return X, y.reshape(-1, 1)


true_w = torch.tensor([2.0, -3.4])
true_b = 4.2

features, labels = synthetic_data(true_w, true_b, 1000)

dataset = TensorDataset(features, labels)
data_iter = DataLoader(dataset, batch_size=10, shuffle=True)

模型：

class LinearRegression(nn.Module):
    def __init__(self):
        super().__init__()
        self.linear = nn.Linear(2, 1)

    def forward(self, X):
        return self.linear(X)


model = LinearRegression()

2. loss 和 optimizer

loss_fn = nn.MSELoss()
optimizer = torch.optim.SGD(model.parameters(), lr=0.03)

注意：

nn.MSELoss() 默认 reduction="mean"

所以 loss 本身已经是标量平均损失，通常直接：

loss.backward()

不需要再写：

loss.sum().backward()

3. 训练循环

for epoch in range(num_epochs):
    for X, y in data_iter:
        y_hat = model(X)
        loss = loss_fn(y_hat, y)

        optimizer.zero_grad()
        loss.backward()
        optimizer.step()

翻译：

拿一个 batch
前向预测
计算损失
清空旧梯度
反向传播得到新梯度
更新参数

训练后：

print(model.linear.weight.detach())
print(model.linear.bias.detach())

应接近：

weight ≈ [[2.0, -3.4]]
bias   ≈ [4.2]

六、nn.Sequential 与 nn.ReLU

1. Sequential 的作用

如果模型只是简单地一层接一层，可以写：

model = nn.Sequential(
    nn.Linear(784, 256),
    nn.ReLU(),
    nn.Linear(256, 10)
)

它表示：

X -> Linear(784, 256) -> ReLU -> Linear(256, 10) -> output

nn.Sequential 本身也是 nn.Module，所以支持：

model(X)
model.parameters()
model.named_parameters()
model.train()
model.eval()

2. 哪些层有参数

for name, param in model.named_parameters():
    print(name, param.shape)

可能输出：

0.weight torch.Size([256, 784])
0.bias   torch.Size([256])
2.weight torch.Size([10, 256])
2.bias   torch.Size([10])

没有 1.weight，因为第 1 层是：

nn.ReLU()

ReLU 没有可学习参数。

3. ReLU 的意义

ReLU 定义：

ReLU(x) = max(0, x)

负数变 0，正数保持不变。

如果只堆：

Linear -> Linear -> Linear

整体仍然等价于一个线性层。

加入 ReLU 后：

Linear -> ReLU -> Linear

模型才有能力拟合更复杂的非线性关系。

七、分类任务中的 CrossEntropyLoss

1. 官方 loss 接收 logits

分类模型最后一层通常输出：

logits.shape = (batch_size, num_classes)

例如 Fashion-MNIST：

logits.shape = (batch_size, 10)
y.shape      = (batch_size,)

训练时：

loss_fn = nn.CrossEntropyLoss()
loss = loss_fn(logits, y)

不要先写：

probs = torch.softmax(logits, dim=1)
loss = loss_fn(probs, y)

因为 nn.CrossEntropyLoss 内部已经包含：

log_softmax + NLLLoss

也就是把 softmax 和交叉熵相关步骤合在一起了。

2. 标签是类别索引，不是 one-hot

正确：

y = torch.tensor([0, 2, 1, 9])

表示：

第 1 个样本类别 0
第 2 个样本类别 2
第 3 个样本类别 1
第 4 个样本类别 9

常见输入：

logits: (batch_size, num_classes)
y:      (batch_size,)

3. accuracy 不需要 softmax

预测类别：

pred = logits.argmax(dim=1)

准确率：

acc = (pred == y).float().mean()

不需要先 softmax，因为 softmax 不改变每行最大值所在的位置。

注意：

argmax(dim=1)：对每个样本，在类别维度上找最大分数
argmax(dim=0)：跨 batch 比较，不是分类预测想要的语义

八、分类训练骨架

1. Fashion-MNIST MLP 模型骨架

model = nn.Sequential(
    nn.Flatten(),
    nn.Linear(784, 256),
    nn.ReLU(),
    nn.Linear(256, 10)
)

shape 流动：

X:              (batch_size, 1, 28, 28)
Flatten:        (batch_size, 784)
Linear 1:       (batch_size, 256)
ReLU:           (batch_size, 256)
Linear 2:       (batch_size, 10)

最后输出：

logits: (batch_size, 10)

2. 单个 batch 训练步骤

logits = model(X)
loss = loss_fn(logits, y)

optimizer.zero_grad()
loss.backward()
optimizer.step()

3. 一个 epoch 的训练骨架

for epoch in range(num_epochs):
    model.train()

    for X, y in train_iter:
        logits = model(X)
        loss = loss_fn(logits, y)

        optimizer.zero_grad()
        loss.backward()
        optimizer.step()

4. 评估骨架

model.eval()

correct = 0
total = 0

with torch.no_grad():
    for X, y in test_iter:
        logits = model(X)
        pred = logits.argmax(dim=1)

        correct += (pred == y).sum().item()
        total += y.numel()

test_acc = correct / total

注意：

model.eval()：切换评估模式，影响 Dropout / BatchNorm 等层
torch.no_grad()：关闭梯度记录，节省内存和计算

二者不是一回事，正式评估时通常一起使用。

九、今天容易踩的坑

1. 忘记 super().__init__()

自定义 nn.Module 时要写：

super().__init__()

否则子层和参数注册可能出问题。

2. 在 forward 里新建层

错误写法：

def forward(self, X):
    linear = nn.Linear(2, 1)
    return linear(X)

问题：

每次 forward 都创建新参数
optimizer 管不到稳定的一组参数
训练无法正常收敛

正确做法：

def __init__(self):
    super().__init__()
    self.linear = nn.Linear(2, 1)

def forward(self, X):
    return self.linear(X)

3. 层没有挂到 self 上

错误：

linear = nn.Linear(2, 1)

如果没有 self.linear，PyTorch 不会把它登记为模型子层。

4. 给 CrossEntropyLoss 提前 softmax

错误：

probs = torch.softmax(logits, dim=1)
loss = nn.CrossEntropyLoss()(probs, y)

正确：

loss = nn.CrossEntropyLoss()(logits, y)

5. 标签 shape 写错

CrossEntropyLoss 常见输入：

logits: (batch_size, num_classes)
y:      (batch_size,)

标签通常是类别索引，不是 one-hot。

6. argmax 维度写错

分类预测：

pred = logits.argmax(dim=1)

不是：

pred = logits.argmax(dim=0)

7. 混淆 model.eval() 和 torch.no_grad()

model.eval() 改变模型中某些层的行为
torch.no_grad() 关闭梯度记录

它们解决的问题不同。

十、今天的核心对应关系

手写 w / b
-> nn.Linear 内部的 weight / bias
手写 net(X, w, b)
-> forward / model(X)

-> model.parameters()
手写 squared_loss
-> nn.MSELoss()
手写 softmax + cross entropy
-> nn.CrossEntropyLoss()
手写 SGD
-> torch.optim.SGD
手写 param.grad.zero_()
-> optimizer.zero_grad()
