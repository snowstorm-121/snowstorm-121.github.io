# 首页时钟歌词组件设计

**日期：** 2026-07-18  
**状态：** 已确认，待实现

## 目标

将首页右侧当前的 `memory-aperture` 替换为用户参考图同构的时钟组件，并将用户已提供的九首 LRC 歌词与本地音乐播放器同步。社交入口图标同步放大，使图形在圆形边框中占据更饱满的视觉比例。

## 视觉与布局

- 桌面端将中部标题、搜索和短句整体向左移动并适度收窄，为右侧组件预留独立视觉列；不改动左侧头像、简介及播放器布局。
- 右侧组件保持绝对定位并增大至约 220–250px 宽的卡片尺度，且与搜索栏、标题之间留出可见空隙；窄屏端回流至内容区。
- 外层为圆角磨砂玻璃卡片；顶部显示实时本地 `HH:MM:SS` 数字时钟，中部显示中文日期及星期。
- 下部使用青绿色及荧黄色分段同心轨道环，整体结构、比例和冷色玻璃质感以用户提供的参考图为准。
- 圆环不再使用光圈叶片、刻度或旧的短句按钮视觉。
- 社交圆按钮维持原位置和链接，图标尺寸提升至圆框可读范围内的最大安全尺寸。

## 交互

- 时钟每秒更新，日期以访问者的本地时间和时区格式化。
- 非减少动态模式下，轨道环以低速持续旋转，外圈与内圈采用不同方向和速度；时钟数字在秒数变化时产生一次极短的亮度呼吸。
- 鼠标进入卡片后，卡片明显抬升并产生受限的三维视差，轨道转速提升并出现分段追光，玻璃高光横向扫过、青绿色光晕向外扩散；歌词窗以弹性缓动展开为三行。
- 鼠标移动时，圆环、歌词窗和高光根据指针位置产生不同幅度的景深偏移；不改变卡片在布局中的占位，避免与中部内容碰撞。
- 鼠标移出平滑恢复；`prefers-reduced-motion: reduce` 时关闭连续旋转与视差，但保留时间与歌词更新。
- 点击卡片不改变播放器状态，避免与现有播放按钮产生职责重叠。

## 歌词区域

- 组件默认保留完整时钟、日期与圆环；开始播放时，在圆环下方/卡片底部的半透明歌词窗展示当前歌曲名和当前一句歌词。
- 鼠标悬停时歌词窗扩展为“上一句 / 当前句 / 下一句”三行，当前句高亮；移出后收回为单行。
- 歌词随现有 `profileAudio.currentTime` 驱动，无网络请求。暂停时保持最后一句；切歌或歌曲结束时立即重置到对应歌曲的首句。
- 歌词文件来自用户目录 `/Users/yyy/Learning_material/music/LRC` 中的九个 `.lrc` 文件，并在实现时复制至仓库 `assets/music/lyrics/`。LRC 解析仅支持标准 `[mm:ss.xx]` 时间戳；忽略元数据行与空文本行。
- 若任一歌词文件加载失败或没有可匹配的时间点，组件仅展示曲名和“歌词暂不可用”，不影响播放与时钟。

## 数据接口

- 扩展现有 `tracks` 项，添加稳定 ASCII 文件名的 `lyrics` 路径。
- 页面初始化时预取或按需读取 LRC，解析为 `{ time: number, text: string }[]`；只缓存已加载曲目的解析结果。
- 新组件通过已存在的播放器状态（当前曲目索引、`profileAudio` 事件）获取数据，不创建第二套音频状态。

## 验收与测试

- 自动测试断言：新时钟组件、歌词容器、九条 LRC 路径、实时更新时间函数、播放器时间更新钩子、悬停/减少动态 CSS 均存在。
- 在桌面宽度验证搜索栏不与组件重叠；在窄屏验证组件回流、不遮挡个人信息或播放器。
- 手动验证：九首歌曲均能显示正确曲名和随播放时间变化的歌词；悬停展开三行歌词；微信入口仍可二次点击关闭。
