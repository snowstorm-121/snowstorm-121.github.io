# 紧凑黑胶播放器与固定中间栈设计

**日期：** 2026-07-20
**状态：** 已确认，待实施

## 目标

将当前偏大的右侧黑胶组件收敛为参考站风格的紧凑信息卡，并让搜索栏与动态文字成为不随内容位移的稳定中间列。

## 布局与尺寸

- 桌面右侧播放器宽度为 `clamp(304px, 25vw, 348px)`；唱盘直径为 `clamp(148px, 12vw, 164px)`。
- 唱片标签直径为唱盘的 70%，保留沟槽与银色窄边，不让黑色唱盘环压过封面主体。
- 桌面 Hero 使用明确的左资料、中间固定栈、右播放器三块安全区；中间列的宽度、左边界、两行文字高度不由动态文字内容决定。
- 当三列无法保持安全间距时，播放器回流至资料下方；1024px 与 320px 不出现组件重叠或裁切。

## 唱针与交互

- 暂停时，唱针处于唱片外侧的待机位置。
- 播放时，唱针绕既有轴心旋入唱片右上部的可播放沟槽；唱片持续旋转。
- 保留受限的悬停浮起、倾斜和反光；减少动态时保持静态。

## 社交入口

- 恢复图形化的 GitHub、邮件、微信、QQ、YouTube、X 图标外观。
- 图标必须以内联 SVG 实现，不重新加入任何 CDN、字体图标或其他外部运行时资源。

## 验收

- 1440px：播放器紧凑，封面明显大于原版本；搜索与文字位置在短句/长句间不变；三块区域不互相遮挡。
- 1024px、320px：所有内容在普通流中回流，无横向溢出、遮挡或被裁切的唱针/歌词。
- 播放与暂停在唱针位置上有明确差异；减少动态仍关闭动画。
- 社交图标为图形 SVG，且页面无外部图标运行时请求。
