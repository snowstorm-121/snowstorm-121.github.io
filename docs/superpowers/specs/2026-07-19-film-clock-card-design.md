# 首页胶片播放时钟卡片设计

**日期：** 2026-07-19  
**状态：** 已确认，待用户审阅规格文件

## 目标

在不改变左侧个人资料、原生音乐播放器和社交入口的前提下，解决桌面端中部标题/搜索与右侧组件互相挤压的问题，并把右侧时钟歌词卡片改造成与夜海主页一致的“毛玻璃胶片播放机”。

## 布局

- 桌面端将 `.hero-main` 固定在头像资料右侧、时钟卡片左侧的安全列中；它不再由动态文字的长度决定宽度或右边界。
- 桌面端标题与搜索共用这一安全列：标题缩小并保持 `STILL, I GO ON` 单行，搜索栏宽度固定跟随安全列，不得与卡片重叠，也不得压入左侧头像/简介区域。
- 窄屏端继续沿用现有回流布局：组件进入普通文档流，标题和搜索恢复 100% 宽度。

## 卡片视觉

- 保留卡片 220–250px 的整体尺度与实时时间、中文日期、LRC 歌词功能。
- 卡片采用深蓝灰半透明毛玻璃：提高背景透视与模糊层次，弱化当前厚重黑色边框；使用冷青边缘光和低强度暖金反射，与夜海背景的海面/地平线色调衔接。
- 卡片外框增加低调胶片语言：细窄齿孔、边缘编号刻度、颗粒噪点和轻微划痕，仅作为装饰，不能降低文字可读性。
- 中部圆环成为“播放卷盘”：保留青绿/荧黄分段轨道，并增加同心进度细环。
- 新增三类信息：`NOW PLAYING` 状态标记、歌曲名/歌手，以及 `当前时间 / 总时长` 的播放时间读数；都由既有 `TRACKS` 与唯一 `profileAudio` 驱动，不新增音频实例或外部请求。
- 新增迷你均衡器装饰，只有在现有播放器实际播放时显示活跃状态。

## 交互状态

- **静止：** 卡片低对比、卷盘低速旋转、均衡器安静；不遮挡页面主视觉。
- **悬停/聚焦：** 卡片明显上浮，外沿冷光与暖色反射增强；胶片高光横扫，卷盘加速并出现追光，歌词展开为三行。
- **指针移动：** 通过既有受限 CSS 变量驱动卡片、卷盘、高光、字幕条的不同深度偏移；视觉上必须明显区别于静止状态，但不改变布局占位或碰撞周边内容。
- **点击/触摸：** 沿用现有 `aria-expanded`/`.is-expanded` 披露状态，可固定展开详细歌词与播放信息，再次激活收起；不得改变播放/暂停状态。
- **播放状态：** 播放时均衡器与进度环活跃；暂停/结束时视觉回落，歌词仍保留在最后匹配句。
- **减少动态：** 保留时间、歌词、播放进度和可点击披露；关闭卷盘连续旋转、均衡器动画、高光扫动、颗粒闪帧与鼠标视差。

## 数据与可访问性

- 复用现有本地 LRC 缓存、歌词同步、时间更新和播放器事件；不添加第三方库、外部歌词服务或第二个 `Audio` 实例。
- `NOW PLAYING`、进度和均衡器均为补充信息；歌词 live region 继续仅在内容变化时更新，避免重复播报。
- 装饰性的齿孔、颗粒、划痕、卷盘和均衡器使用 `aria-hidden`；按钮语义与 `aria-expanded` 保持可用。

## 验收

- 1440px 桌面端：标题保持单行，标题/搜索右侧与卡片之间存在可见间隙，左侧头像与简介不被中部内容遮挡。
- 动态文字改变时，搜索栏宽度和右边界保持稳定。
- 卡片呈现夜海毛玻璃和胶片元素；显示曲目信息、进度、时间码和播放状态。
- 悬停、指针移动、播放、暂停与点击展开之间有清晰、可观察的状态区别。
- 窄屏回流、减少动态、LRC 歌词同步、原生播放器与社交入口保持可用。
