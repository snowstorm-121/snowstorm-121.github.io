# Midnight Observatory 个人主页整页重设计

**日期：** 2026-07-24
**状态：** 用户已确认设计方向，待实施计划
**主题：** Apple 风格深色液态玻璃、滚动叙事、全局音乐与固定动态文字

## 1. 设计目标

将当前首页从“多个独立卡片排列”重构为一段连续的夜海观测旅程。页面要有 Apple 产品页式的秩序、留白、材质层次和克制动画，同时保留个人主页原有的内容辨识度。

最终体验应满足：

- 第一眼安静、深邃、有高级感，而不是依赖大量发光和粒子制造热闹。
- 页面各章节具有清晰的视觉节奏，滚动时像在进入不同深度的夜海空间。
- 搜索、动态文字、个人资料、教育信息、作品、社交入口和音乐彼此融合，不再像互不相关的组件集合。
- 音乐是贯穿页面的氛围层，不占用一个独立的全屏章节。
- 动画帮助理解层级和状态，不影响阅读、不造成布局移动。

## 2. 必须保留的现有能力

- 左侧头像、个人简介和教育信息。
- Google 搜索功能。
- 现有动态文字内容与轮播顺序。
- 动态文字正常模式下“逐字删除旧句，再逐字输入新句”的行为。
- 全部九首本地 MP3、LRC 和原创封面。
- 唯一的 `profileAudio`；不得创建第二个 `audio` 元素。
- 歌词 `aria-live` 区域及播放、暂停、切歌、进度同步能力。
- 现有作品入口和社交链接。
- 已确认的 GitHub、邮箱、QQ、微信、YouTube 等图标语义。

## 3. 页面叙事结构

整页由四个连续章节和一个全局音乐层构成。

### 3.1 Origin / 首屏

首屏负责建立个人气质和页面主题：

- 左侧或视觉主轴区域显示简洁的个人宣言。
- 搜索栏位于宣言下方的固定位置。
- 动态文字位于搜索栏下方，保持原有逐字切换。
- 右侧使用抽象“月光观测装置”：由半透明圆环、柔和折射、夜海光谱和空间薄雾组成，不再显示大型黑胶播放器卡片。
- 底部提供轻量的滚动提示，点击后进入下一章节。

月光装置是独立的氛围交互，不承载音乐控制或其他业务状态。它使用原生按钮语义并提供明确的辅助名称：点击、触摸或键盘激活时只释放一次柔和光波；支持精确指针的设备还可产生细微景深。反馈结束或指针离开后恢复静止，不持续抢夺注意力。

### 3.2 Identity / 个人信息

个人资料与教育信息使用两块错落但对齐的玻璃面板：

- 主面板展示头像、姓名、简介和当前关注方向。
- 次面板展示教育经历和时间信息。
- 两块面板之间通过一条低亮度光线建立联系，而不是使用传统时间轴粗线。
- 桌面端面板可以有轻微空间层次；移动端按正常文档流纵向排列。

信息始终直接可读。悬停和聚焦只增加材质反馈，不隐藏必要文字，也不要求用户操作才能看到核心内容。

### 3.3 Selected Archive / 作品档案

作品区使用不对称 Bento 布局，但避免密集的小方块：

- 一个主作品占据更大的展示面积。
- 其余作品以一至两块辅助面板排列。
- 每张卡片包含标题、简短说明、时间或分类以及明确的访问入口。
- 页面在进入作品章节时，由深蓝黑过渡到更明亮的银蓝色玻璃环境，再在离开时回到深色。

桌面端悬停或键盘聚焦时，卡片产生轻微浮起、局部折射和内容层次变化。点击作品的预览区域可在原卡片附近展开补充说明；点击明确的访问按钮才打开目标链接，避免整个卡片同时承担多种不清晰行为。

### 3.4 Connection / 社交联系

最后一屏收束页面：

- 使用简短邀请语和原有社交入口。
- 图标保持清晰的品牌辨识度，不重新设计成难以识别的抽象符号。
- 社交按钮在桌面端有轻微磁吸和玻璃内浮效果；触屏设备使用按压反馈。
- QQ 只执行链接跳转，不弹出 QQ 号。
- 微信点击后打开小型联系方式面板，显示微信号并提供复制按钮；点击面板外、关闭按钮或按 `Escape` 均可关闭。不得伪装成一个不存在或不可用的直接聊天链接。

## 4. 搜索栏与动态文字的固定布局

这一部分是强制几何约束，优先级高于视觉动画。

### 4.1 独立布局槽位

搜索栏与动态文字必须放在同一个固定宽度的叙事列中，但分别占用两个独立的网格行：

1. 搜索栏行具有固定高度。
2. 搜索栏与动态文字之间具有固定间距。
3. 动态文字行具有固定的两行高度。
4. 动态文字节点的内容变化只能发生在自己的槽位内部。

搜索栏不得由动态文字容器包裹，不得使用随内容尺寸参与垂直居中的共同父容器，也不得通过测量文字高度重新计算位置。

### 4.2 动态文字行为

- 保留现有句子、顺序和切换节奏。
- 正常模式：旧句从最后一个字符开始逐字删除；删除完成后，新句逐字输入。
- 删除阶段不先清空整个节点，不出现整句突然消失。
- 动态文字最多显示两行，固定行高与固定最小/最大块尺寸一致。
- 对超出两行安全长度的文本，通过已确定的字号、列宽和换行策略解决，不允许推动搜索栏或相邻章节。
- 光标属于动态文字槽位，不影响文本测量和布局宽度。

### 4.3 减少动态

在 `prefers-reduced-motion: reduce` 下：

- 关闭逐字删除、逐字输入和光标闪烁。
- 每次更新直接替换为完整句子。
- 仍保留相同的两行固定高度和相同搜索栏位置。

## 5. 全局音乐体验

### 5.1 紧凑音乐 Dock

音乐播放器不再占用首屏右侧，也不新增专门的音乐章节。它作为全局悬浮 Dock 固定在页面底部安全区域：

- 收起状态显示当前封面、歌曲名、播放/暂停和展开入口。
- 播放状态使用低强度呼吸光和有限的进度反馈。
- 切歌时封面、歌曲名和颜色连续过渡，不让 Dock 宽度突然变化。
- Dock 不遮挡正文、社交按钮或移动端浏览器底栏。
- 页面正文为固定 Dock 预留与其实际高度和设备安全区一致的底部空间。

顶部导航可显示非常简短的播放状态，但不复制完整控制。

### 5.2 展开音乐面板

点击 Dock 的展开入口后：

- 桌面端从 Dock 上方展开为紧凑玻璃面板。
- 移动端使用底部抽屉，并尊重设备安全区。
- 展示当前封面、歌曲信息、上一首/播放/下一首、进度、歌曲列表以及上一行/当前行/下一行歌词。
- 当前歌词保持 `aria-live="polite"`。
- 支持关闭按钮、`Escape` 和点击面板外关闭。
- 展开与关闭不重建音频元素，不中断当前歌曲。
- 关闭后将焦点返回触发展开的控件。
- 320px 窄屏底部抽屉的最大高度不超过 `100dvh`，扣除安全区后让歌曲列表与歌词区域在面板内部滚动，关闭按钮和播放控制始终可见。

### 5.3 音乐数据流

现有 `TRACKS` 继续作为九首歌曲的唯一数据源。用户操作只更新当前曲目状态；`profileAudio` 负责实际播放，时间更新驱动歌词索引和进度显示。导航状态、Dock、展开面板和环境光都读取同一播放状态，不各自维护互相冲突的副本。

浏览器禁止自动播放时，页面保持暂停并等待用户首次操作，不显示错误弹窗。LRC 加载失败时仍允许播放歌曲，并在歌词区域显示简短的不可用状态。封面不可用时使用 CSS 生成的无文字渐变占位，不请求远程图片。

## 6. 增强叙事型交互

### 6.1 滚动叙事

- 章节进入视口时，内容以轻微上移和透明度变化出现。
- 当前章节同步更新顶部导航和右侧细线进度轨道。
- 进度轨道的节点可点击并平滑前往对应章节。
- 背景色温、雾化程度和主光方向随章节连续变化，不做突兀的整屏闪白。
- 未加载 JavaScript 或观察器不可用时，所有内容默认可见，导航锚点仍可工作。

### 6.2 液态玻璃响应

仅在支持精确指针且未开启减少动态时启用：

- 指针经过主要玻璃表面时，表面高光在有限范围内缓慢跟随。
- 卡片只允许极小角度倾斜，不跟随页面全局指针持续旋转。
- 指针离开后使用短缓动回到中性状态。
- 多张卡片不会同时运行高频动画。

玻璃视觉由统一的颜色、透明度、边缘光、内阴影、背景模糊和噪点层构成。不得为每个组件创建互不一致的“玻璃配方”。

### 6.3 搜索与按钮反馈

- 搜索框聚焦时逐渐点亮边缘和内部光带。
- 合法提交时产生一次短流光；空搜索保持在页面内并给出可访问的轻量提示。
- 主要按钮支持清晰的悬停、按压、键盘聚焦状态。
- 桌面端社交按钮可有轻微磁吸位移，最大位移必须受限；移动端完全不启用磁吸。

### 6.4 作品探索

- 卡片悬停时改变局部折射中心，并显示补充信息入口。
- 展开预览时从卡片自身位置延展，关闭后回到原位置。
- 同一时间最多展开一个作品。
- `Escape` 可关闭预览；焦点在展开和关闭后保持可预测。

### 6.5 音乐与环境联动

- 播放时 Dock 与邻近环境光使用极低强度的呼吸效果。
- 切歌时页面环境色在统一深海蓝、月光青和少量金色范围内调整。
- 当前歌词逐行切换时，背景光谱在同一受限色域内缓慢、低强度地变化；它不改变文字对比度，也不改变任何布局尺寸。
- 歌词逐行切换只使用短淡入或轻移，不放大、不弹跳。
- 音频暂停后，连续呼吸效果停止，唱片封面不持续旋转。

### 6.6 静置状态

在用户连续 20 秒没有滚动、点击、触摸或键盘输入时，页面进入“夜海呼吸”状态：

- 降低而非增加动画强度。
- 减慢背景光的呼吸节奏。
- 不自动播放音乐、不自动切换章节、不隐藏内容。
- 用户再次操作时立即恢复正常响应。

## 7. 动画原则与减少动态

动画分为三类：

1. **状态动画：** 播放、展开、聚焦和章节激活，必须保留明确的静态状态差异。
2. **过渡动画：** 内容进入、卡片展开和切歌，可在减少动态下改为短淡入或直接切换。
3. **连续氛围动画：** 呼吸光、视差、磁吸、倾斜和高光跟随，在减少动态下全部关闭。

`prefers-reduced-motion: reduce` 下还必须：

- 关闭动态文字逐字动画。
- 关闭平滑滚动、视差、卡片倾斜、磁吸和连续呼吸。
- 关闭歌词驱动的背景光谱过渡和月光装置光波动画；月光按钮仍保留清晰的静态按压反馈。
- 关闭月光装置的持续运动。
- 保留歌曲切换、歌词内容更新、面板开关和导航状态。
- 不因关闭动画改变任何组件的最终位置和尺寸。

## 8. 响应式布局

### 8.1 宽屏

- 首屏采用清晰的文字叙事区和月光装置区，两者具有独立安全边界。
- 搜索和动态文字的固定列不得进入右侧装置范围。
- Identity 使用错落双面板；Archive 使用不对称布局。
- 音乐 Dock 保持紧凑，不覆盖章节进度轨道。

### 8.2 约 1024px

- 首屏改为单主列加缩小装置，或让装置进入文字下方的普通文档流。
- 不使用负边距强行维持桌面构图。
- Identity 和 Archive 根据可用空间变为两列或单列。
- 搜索栏仍保持固定槽位，动态文字变化不影响其位置。

### 8.3 320px 窄屏

- 所有内容单列排列，不出现水平滚动。
- 月光装置缩为背景氛围，不抢占正文高度。
- 动态文字保留固定两行高度，并使用移动端安全字号。
- 音乐 Dock 显示封面、截断后的歌曲名和播放/展开控制。
- 展开音乐使用底部抽屉；控制按钮满足触摸目标尺寸。
- 右侧章节轨道隐藏或改为顶部短进度线。

## 9. 可访问性与输入方式

- 所有可点击元素使用原生链接或按钮，不给纯装饰节点添加虚假交互。
- 键盘可以操作导航、搜索、作品预览、音乐控制和社交入口。
- `:focus-visible` 状态在深色和浅色章节都清晰可见。
- 展开控件维护正确的 `aria-expanded` 和目标关系。
- 装饰性光线、折射和背景图层不进入辅助技术阅读顺序。
- 当前歌词继续使用礼貌播报，不能在每个音频时间更新事件中重复朗读相同歌词。
- 颜色不是播放、聚焦或当前章节状态的唯一表达方式。

## 10. 技术边界

- 不增加 npm 包、前端框架、图标运行时或动画依赖。
- 不使用远程图片、远程字体、CDN 或第二个音频实例。
- 使用现有本地资源和浏览器原生 CSS/JavaScript 能力。
- 可为了可维护性将整页样式和交互脚本从 `index.html` 拆分为页面专用本地文件，但不得改变静态部署方式。
- 只重构首页相关结构、样式、脚本和测试；不顺手改造无关归档页面。
- 原有链接、音乐路径和个人内容以当前仓库为准，不在视觉重构中擅自改写。

## 11. 验收标准

### 11.1 功能

- 页面恰好只有一个 `<audio id="profileAudio">`，脚本中不存在 `new Audio()` 或其他额外音频实例。
- 九首本地歌曲、LRC 和封面均可继续使用。
- 播放、暂停、上一首、下一首、歌曲列表、进度和歌词同步正常。
- 搜索提交、作品链接和社交入口正常。
- QQ 不弹出号码；微信行为与设计说明一致。

### 11.2 布局

- 动态文字在短句、长句、删除和输入阶段均不改变搜索栏的屏幕位置。
- 动态文字始终占用固定两行高度。
- 宽屏、1024px 和 320px 下没有组件遮挡、正文裁切或横向溢出。
- 音乐 Dock 和展开面板不遮挡关键操作。

### 11.3 视觉与交互

- 四个章节具有统一的深色液态玻璃语言和清晰的明暗节奏。
- 玻璃高光、章节转场、作品预览、音乐 Dock 和环境联动均有明确但克制的反馈。
- 动画结束后组件回到确定状态，不存在漂移或累积变换。
- 键盘和触摸可以完成与鼠标等价的核心操作。

### 11.4 减少动态

- 关闭逐字动画、平滑滚动、视差、倾斜、磁吸和所有连续呼吸动画。
- 歌曲和歌词仍正常更新。
- 所有内容位置、尺寸和可访问状态与普通模式一致。

### 11.5 验证

- 扩展 Node 静态测试，覆盖唯一音频、本地资源、固定搜索/文字结构、歌词 live region、交互状态和减少动态约束。
- 实施时遵循测试驱动：先增加失败测试，再实现，再运行完整测试通过。
- 浏览器分别检查桌面宽屏、1024px、320px 和减少动态模式；跨越一次完整的删除与输入周期，在短句、长句、删除至空和重新输入阶段采集搜索栏的 `getBoundingClientRect()`，断言 `top`、`left`、`width`、`height` 与周期开始时相比最多只有 1px 渲染误差。
- 检查键盘操作、`Escape` 关闭、焦点返回、触摸目标和无横向滚动。
